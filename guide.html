프로젝트 개요

🎯 MOODSPOT이란?

감정을 기반으로 하루를 설계해주는 AI 경험 큐레이터 모바일 앱. 감정 입력 → AI 분석 → 문화 처방 → 장소 추천 → 코스 생성 → 지도 탐색까지 끊김 없는 UX 흐름을 제공한다.

📱 플랫폼

  • iOS 앱 (App Store 배포)
  • Android 앱 (Google Play 배포)
  • 웹 랜딩페이지 (홍보용)
  • 관리자 대시보드 (내부용)

👥 타겟 사용자

  • 20-35세 도시 직장인/학생
  • 감정적 피로를 느끼는 현대인
  • 문화/여가 활동을 즐기는 사람
  • 새로운 경험을 원하지만 결정 못하는 사람

기술 스택 — 무엇으로 만들까?

💡
선택 기준 수정이 쉽고, 한국 서비스에 최적화되어 있으며, 팀 규모가 작을 때 빠르게 개발 가능한 스택을 선택했습니다.

📱 프론트엔드 (앱)

⚛️
React Native
iOS + Android 동시 개발
하나의 코드로 iOS/Android 동시 배포. JavaScript 기반이라 웹 개발자도 빠르게 적응 가능. Expo 사용 시 더 빠른 개발.
✓ 추천
🐦
Flutter
대안
Dart 언어 학습이 필요하지만 성능이 뛰어남. React Native에 익숙하지 않다면 고려할 수 있음.
대안

🖥️ 백엔드 (서버)

🟢
Node.js + NestJS
API 서버
TypeScript 기반, 구조화된 프레임워크. 모듈 단위로 기능 분리되어 유지보수 매우 쉬움. REST API + WebSocket 모두 지원.
✓ 추천
🐍
Python + FastAPI
AI 처리 전용 서버 (보조)
AI/ML 작업에 최적. Claude API 연동, 감정 분석 처리를 별도 Python 서버로 분리하면 성능 향상.

서버 구성 — 어떻게 연결할까?

🚀
단계별 확장 전략 (소규모 → 중규모 → 대규모) 처음부터 큰 서버를 구축할 필요 없습니다. Supabase + Vercel로 시작하고, 사용자가 늘면 AWS로 이전하세요.

🥚 1단계: 초기 (0~1,000명)

가장 빠르고 저렴하게 시작하는 방법

# 초기 스택 구성

프론트엔드/랜딩:  Vercel (무료)
관리자 대시보드:  Vercel (무료)
백엔드 API:       Railway 또는 Render ($5~20/월)
데이터베이스:     Supabase (무료 티어 → Pro $25/월)
파일 저장:        Supabase Storage (무료)
AI:               Claude API (사용량 기반)
지도:             카카오맵 API (무료)

# 총 예상 비용: 월 $30~50

🐣 2단계: 성장기 (1,000~10,000명)

안정성과 성능을 높이는 단계

# 성장기 스택 구성

프론트엔드:  Vercel Pro ($20/월)
백엔드:      AWS EC2 t3.small ($15~20/월)
DB:          AWS RDS PostgreSQL ($25~50/월)
캐시:        AWS ElastiCache Redis ($15/월)
파일:        AWS S3 ($5~10/월)
CDN:         CloudFront ($5~10/월)

# 총 예상 비용: 월 $100~150

📡 API 연결 흐름

# 앱 → 서버 통신 구조

앱 (React Native)
  ↓ HTTPS 요청
API Gateway (NestJS)
  ├── /auth          → 인증/로그인
  ├── /emotion       → 감정 분석 (Claude API 호출)
  ├── /recommend     → 장소/코스 추천
  ├── /places        → 카카오맵 API 연동
  ├── /courses       → 코스 CRUD
  └── /community     → 커뮤니티 피드

# WebSocket (실시간 AI 대화)
앱 ←→ Socket.IO ←→ AI 대화 서버
⚠️
환경변수 관리 필수! Claude API 키, 카카오 API 키, DB 비밀번호는 절대 코드에 직접 넣지 말고 .env 파일 + 클라우드 시크릿 관리자(AWS Secrets Manager 또는 Vercel Environment Variables)를 사용하세요.

AI 시스템 구현

🧠 Claude API 활용 전략

MOODSPOT의 모든 AI 기능은 Claude API(Anthropic) 하나로 구현 가능합니다.

// 감정 분석 프롬프트 예시 (NestJS)
const response = await anthropic.messages.create({
  model: 'claude-sonnet-4-6',
  max_tokens: 1024,
  system: `당신은 공감 능력이 뛰어난 감정 분석 AI입니다.
사용자의 감정을 분석하고 문화 경험을 처방합니다.
응답은 반드시 JSON 형식으로 반환하세요.`,
  messages: [{
    role: 'user',
    content: `사용자 입력: "${userInput}"

다음 JSON 형식으로 분석하세요:
{
  "emotion": "지침|설렘|답답함|여유",
  "intensity": 1-10,
  "empathy_message": "공감 메시지",
  "prescription": ["미디어아트", "독립서점"],
  "insight": "추천 이유"
}`
  }]
});

🎯 AI 기능별 활용

  • 감정 분석: 자연어 → 감정 태그 변환
  • 공감 대화: 위로 메시지 생성
  • 문화 처방: 감정 → 추천 카테고리
  • 코스 생성: 장소 배열 + 동선 설명
  • AI Insight: 추천 이유 텍스트 생성

💡 비용 절감 팁

  • Prompt Caching으로 시스템 프롬프트 캐싱 (90% 절감)
  • 추천 결과는 Redis에 1시간 캐싱
  • 같은 감정 패턴은 DB에서 재사용
  • claude-haiku-4-5는 단순 분류에 활용

수정하기 쉬운 구조 만들기

🏗️ 폴더 구조 (수정 쉽게)

moodspot/
├── apps/
│   ├── mobile/          # React Native 앱
│   │   ├── src/
│   │   │   ├── screens/ # 화면별 파일 분리
│   │   │   ├── components/ # 재사용 UI 컴포넌트
│   │   │   ├── services/   # API 호출 로직
│   │   │   ├── hooks/      # 커스텀 훅
│   │   │   └── constants/  # 색상, 텍스트 상수
│   └── web/             # Next.js 웹/관리자
│       ├── app/
│       │   ├── (landing)/  # 랜딩페이지
│       │   └── admin/      # 관리자 대시보드
├── packages/
│   ├── api/             # NestJS 백엔드
│   │   └── src/
│   │       ├── modules/ # 기능별 모듈 분리
│   │       │   ├── emotion/
│   │       │   ├── recommend/
│   │       │   ├── places/
│   │       │   └── courses/
│   └── shared/         # 공유 타입/유틸
└── docs/               # 제작 문서

✅ 수정 쉬운 코드 원칙

  • 색상/텍스트는 모두 constants 파일에 정의
  • API URL은 환경변수로 분리 (.env)
  • AI 프롬프트는 별도 파일에서 관리
  • 컴포넌트는 작게 쪼개기 (100줄 이하)
  • 기능별 모듈로 완전 분리 (NestJS)
  • TypeScript 타입 엄격하게 정의

🛡️ 안정성 확보

  • GitHub Actions로 자동 배포 (CI/CD)
  • main 브랜치는 직접 push 금지
  • Sentry로 에러 실시간 감지
  • DB 자동 백업 (매일 1회)
  • 단계적 배포 (feature → dev → prod)
  • API 버전 관리 (/v1/, /v2/)
